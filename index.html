SBS 2011 / Windows Server Trust Relationship Issue After Power Loss

Recently I had a client with an SBS 2011 server that lost power unexpectedly. Following the outage, client PCs were unable to access server resources and displayed messages suggesting the computer or domain relationship may have been compromised.

Once the server had rebooted, the network profile had changed to Unauthenticated Network. Attempting to open the DNS Management console resulted in an Access Denied error.

Symptoms observed:

  • DNS console reporting Access Denied.
  • DCDiag failing connectivity tests.
  • Active Directory Users and Computers still accessible.
  • Clients unable to access server shares and services.

After some investigation I was able to resolve the issue by resetting the secure channel password and then rebooting the server using: 

nltest /sc_change_pwd:domainname.local

Command Description

Changes the password for the trust account of a domain that you specify. If you run NLTest on a domain controller and an explicit trust relationship exists, NLTest resets the password for the inter-domain trust account. Otherwise, it changes the computer account password for the specified domain.

Following the reboot, the network location returned to Domain Network, DNS became accessible again and all client machines were able to access server resources normally.

Comments from administrators on my original blog post indicate that this fix also resolves the same issue on Windows Server 2012 R2.
